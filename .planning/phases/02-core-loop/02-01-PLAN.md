**waveConfig.ts:**
- Import `midiNoteToName` from `../lib/noteUtils`
- Export `LevelConfig` interface: `{ levelIndex: number, allowedNotes: number[], waves: WaveConfig[] }`
- Export `WaveConfig` interface: `{ enemies: EnemySpawnEntry[], spawnIntervalMs: number }`
- Export a helper `buildNoteSpawn(midiNote: number, color: string): EnemySpawnEntry` that creates a note-type spawn entry using `midiNoteToName` for the label
- Export `LEVEL_CONFIGS: LevelConfig[]` with at least 3 levels:
  - Level 0: allowedNotes [60, 64, 67] (C4, E4, G4 — 3 notes). 3 waves. Wave 1: 3 enemies, 2500ms interval. Wave 2: 4 enemies, 2200ms. Wave 3: 5 enemies, 2000ms.
  - Level 1: allowedNotes [60, 62, 64, 65, 67] (C4-G4 — 5 notes). 4 waves. Escalating enemy count 4→5→6→7, intervals 2200→2000→1800→1600ms.

  <action>
Expand the existing GameSlice stub in `src/stores/audioStore.ts` to a full game state machine. Use `immer` middleware from `zustand/middleware/immer` for cleaner enemy array mutations. Import `Enemy` from `../game/enemyTypes` and `EnemySpawnEntry` from `../game/enemyTypes`.

**Important:** The `immer` middleware is bundled with zustand v5 — no npm install needed. Import: `import { immer } from 'zustand/middleware/immer'`. Wrap the store creator with `immer(...)` inside the existing `devtools(...)` wrapper.

**Expand GameSlice interface to include:**
- `gamePhase: 'idle' | 'playing' | 'paused' | 'wave-clear' | 'gameover'`
- `playerHP: number` (default: 5)
- `maxPlayerHP: number` (default: 5)
- `currentWave: number` (0-indexed, default: 0)
- `totalWaves: number` (default: 0)
- `currentLevel: number` (default: 0)
- `enemies: Enemy[]` (default: [])
- `spawnQueue: EnemySpawnEntry[]` (default: [])
- `spawnAccumulator: number` (default: 0)
- `spawnIntervalMs: number` (default: 2000)
- `wrongNoteFlashFrames: number` (default: 0)

**Add GameSlice actions:**
- `startGame: (levelIndex: number) => void` — sets `gamePhase: 'playing'`, resets `playerHP` to `maxPlayerHP`, sets `currentLevel`, loads wave 0 from `LEVEL_CONFIGS[levelIndex]`, populates `spawnQueue` and `spawnIntervalMs` from wave config, resets `currentWave: 0`, sets `totalWaves`, clears `enemies: []`
- `pauseGame: () => void` — sets `gamePhase: 'paused'` (only if currently 'playing')
- `resumeGame: () => void` — sets `gamePhase: 'playing'` (only if currently 'paused')
- `advanceWave: () => void` — if more waves remain: increment `currentWave`, load next wave's `spawnQueue` and `spawnIntervalMs`, set `gamePhase: 'playing'`. If no more waves: set `gamePhase: 'idle'` (level complete — return to menu).
- `spawnEnemy: (entry: EnemySpawnEntry) => void` — calls `buildEnemy(entry)` and pushes to `enemies` array
- `damageEnemy: (id: string, damage: number) => void` — finds enemy by id, reduces `hp` by damage. If `hp <= 0`: set `state: 'dying'`, `defeatedFrames: 12`
- `tickDyingEnemies: () => void` — for each enemy with `state === 'dying'`: decrement `defeatedFrames`. If `defeatedFrames <= 0`: set `state: 'dead'`
- `removeDeadEnemies: () => void` — filter `enemies` to keep only those with `state !== 'dead'`
- `enemyReachedGoal: (id: string) => void` — set enemy `state: 'dead'`, reduce `playerHP` by 1. If `playerHP <= 0`: set `gamePhase: 'gameover'`
- `triggerWrongNote: () => void` — sets `wrongNoteFlashFrames: 8`
- `tickWrongNoteFlash: () => void` — if `wrongNoteFlashFrames > 0`, decrement by 1
- `resetGame: () => void` — resets all GameSlice fields to defaults, sets `gamePhase: 'idle'`

**Update BoundStore type** to include the new GameSlice fields. Keep AudioSlice completely unchanged.

**Wrap store creation with immer:**
Change the store creation from:
```ts

```
to:
```ts

```

Note: The `set` function inside immer-wrapped slices receives a draft that can be mutated directly. The AudioSlice `addEvent` action currently uses the spread/return pattern which still works fine with immer — no changes needed to AudioSlice.

**Update StateCreator generics** for both slices to include the immer middleware type parameter: `[['zustand/devtools', never], ['zustand/immer', never]]`.
