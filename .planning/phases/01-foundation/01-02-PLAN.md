    **src/audio/MidiInput.ts:** Implement per RESEARCH.md Pattern 3:
    - Constructor takes `onNote: (event: NoteEvent) => void` callback
    - `async init(): Promise<boolean>` — calls `navigator.requestMIDIAccess({ sysex: false })`
    - Returns `false` if `requestMIDIAccess` is not available (Safari, old browsers) or permission denied
    - Attaches `onmidimessage` handler to all inputs from `access.inputs`
    - Handles hot-plug via `access.onstatechange` — re-attaches listeners when new devices connect
    - `handleMessage(event: MIDIMessageEvent)`:
      - Extract `[status, note, velocity]` from `event.data`
      - `type = status & 0xF0` to get message type
      - NoteOn: `type === 0x90 && velocity > 0` — emit NoteEvent with type 'NoteOn', source 'midi'
      - NoteOff: `type === 0x80 || (type === 0x90 && velocity === 0)` — emit NoteEvent with type 'NoteOff', source 'midi'
      - CRITICAL: velocity=0 on 0x90 MUST be treated as NoteOff (per MIDI spec, many keyboards use this). Missing this causes stuck notes.
      - Use `midiToNoteName(note)` from types.ts for the noteName field
      - Set `timestamp: performance.now()`

    **src/audio/InputBroker.ts:** Unified event dispatcher:
    - Constructor takes `GameEventBus` instance
    - Method `createNoteHandler(): (event: NoteEvent) => void` — returns a callback that emits the NoteEvent on the GameEventBus via `eventBus.emit('NoteEvent', event)`
    - This callback is what gets passed to MidiInput and AudioPipeline constructors
    - The InputBroker is the single point where both input paths converge — all NoteEvents flow through it

    **src/main.ts updates:** Wire MIDI into the Start button handler:
    - Create InputBroker with the GameEventBus
    - Create MidiInput with `inputBroker.createNoteHandler()`
    - In the Start button click handler: call `midiInput.init()`. Log the result (success/failure) to console
    - MIDI init does NOT require getUserMedia or AudioContext — it can be called immediately, but placing it in the Start handler keeps all initialization in one user gesture

    **src/audio/AudioPipeline.ts:** Implement per RESEARCH.md Pattern 2 and full code example:
    - Import `PitchDetector` from 'pitchy'
    - Import `hzToMidi`, `midiToNoteName`, `NoteEvent` from types.ts

    Constructor takes `onNote: (event: NoteEvent) => void` callback (from InputBroker).

    `async init(): Promise<void>`:
    - MUST be called inside a user gesture handler (the Start button click)
    - Create `AudioContext({ latencyHint: 'interactive' })`
    - Call `this.ctx.resume()` defensively (handle suspended state per Pitfall 1)
    - Call `navigator.mediaDevices.getUserMedia({ audio: true, video: false })`
    - Create `MediaStreamSource` from the stream
    - Create `AnalyserNode` with `fftSize = 2048` (46ms window at 44.1kHz)
    - Connect source -> analyser (do NOT connect analyser to destination — no feedback loop)
    - Create `PitchDetector.forFloat32Array(analyser.fftSize)`
    - Allocate `Float32Array(analyser.fftSize)` buffer
    - Start polling interval at ~30fps (33ms)

    Polling logic (inside `setInterval` at 33ms):
    - Call `analyser.getFloatTimeDomainData(buffer)`
    - Call `detector.findPitch(buffer, ctx.sampleRate)` -> `[frequency, clarity]`
    - **If clarity < 0.9 (CLARITY_THRESHOLD constant at top of file):**
      - If `lastMidi !== null` AND low-clarity has persisted for > 50ms (NOTEOFF_DEBOUNCE constant):
        - Emit NoteOff for lastMidi
        - Set lastMidi = null
      - Track when clarity first dropped (store timestamp)
      - Return early
    - **If clarity >= 0.9:**
      - Reset the low-clarity timestamp tracker
      - Compute `midi = hzToMidi(frequency)`
      - Piano range gate: if midi < 21 or midi > 108, return (reject out-of-range per Pitfall 3)
      - If midi !== lastMidi:
        - If lastMidi !== null: emit NoteOff for lastMidi
        - Emit NoteOn for midi with frequency and clarity fields populated
        - Set lastMidi = midi

    Method `stop(): void` — clears the polling interval, closes AudioContext.

    Constants at top of file (easy to tune per Research Open Question 2):
    ```
    const CLARITY_THRESHOLD = 0.9;
    const NOTEOFF_DEBOUNCE_MS = 50;
    const POLL_INTERVAL_MS = 33; // ~30fps
    ```
