    **vite.config.ts:** Standard Vite config. No special headers needed for Phase 1 (AnalyserNode polling, not AudioWorklet).

    **tsconfig.json:** Enable strict mode. Target ES2020+. Include DOM and DOM.Iterable lib types. Set moduleResolution to bundler.

    **src/audio/types.ts:** Create the NoteEvent interface exactly as specified in RESEARCH.md Pattern 1:
    ```typescript
    export interface NoteEvent {
      type: 'NoteOn' | 'NoteOff';
      midiNote: number;        // 0-127 MIDI note number
      noteName: string;        // e.g., "C4", "A#3"
      source: 'mic' | 'midi';
      frequency?: number;      // Hz — only present for mic source
      clarity?: number;        // 0-1 — only present for mic source
      timestamp: number;       // performance.now()
    }
    ```
    Export `hzToMidi(hz: number): number` using `Math.round(69 + 12 * Math.log2(hz / 440))`.
    Export `midiToNoteName(midi: number): string` using `['C','C#','D','D#','E','F','F#','G','G#','A','A#','B'][midi % 12]` + `Math.floor(midi / 12) - 1` for octave. Verify: MIDI 60 = "C4", MIDI 21 = "A0", MIDI 108 = "C8".
    Export `isWhiteKey(midi: number): boolean` — returns true for C, D, E, F, G, A, B (note % 12 in [0,2,4,5,7,9,11]).

    **src/core/GameEventBus.ts:** Simple typed pub/sub. Methods: `on(event: string, callback)`, `off(event, callback)`, `emit(event, data)`. Use a `Map<string, Set<Function>>` internally. Export a singleton instance. Primary event name: `'NoteEvent'`.

    **src/core/GameLoop.ts:** requestAnimationFrame loop. Constructor takes an `update(dt: number)` and `render()` callback. Use a fixed timestep pattern: accumulate delta time, call update in fixed steps (e.g., 16.67ms = 60fps logic), call render once per frame. Methods: `start()`, `stop()`. Store the rAF handle for cleanup.
  </action>

    - `draw(ctx: CanvasRenderingContext2D, width: number, height: number, highlightedMidi: Set<number>): void`
    - 88 keys: A0 (MIDI 21) to C8 (MIDI 108), 52 white + 36 black
    - White key width = `width / 52`, full strip height
    - Black key width = `whiteKeyWidth * 0.6`, height = `stripHeight * 0.63`
    - Strip background: `#2A2A2A`
    - Inactive white: `#FFFFFF` with `#333` border; inactive black: `#1A1A1A`
    - Active highlight (gold): `#FFD700` — same color for both white and black keys

    - Use `isWhiteKey()` from types.ts to determine key type

    - `init(panel: HTMLElement, eventBus: GameEventBus): void` — subscribes to 'NoteEvent' on the bus

    - Renders each event as a div with: note name, On/Off type, source (mic/MIDI), frequency (or dash), clarity percentage (or dash), timestamp

    - Style event rows with NoteOn in green-ish color, NoteOff in red-ish color for quick scanning
