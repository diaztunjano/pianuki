    Open the app in Chrome (the Vite dev server should already be running).

    **Step 1: Visual layout check**
    1. Verify the game canvas fills the left/center area with a dark blue-gray background and an L-shape path (tan color) with blue "SPAWN" zone at top and green "GOAL" zone at bottom
    2. Verify the keyboard strip at the bottom shows 88 keys (white and black) spanning the full width
    3. Verify the debug panel on the right shows "Waiting for input..." in monospace text
    4. Try resizing the browser window — canvas and keyboard should resize proportionally

    **Step 2: MIDI keyboard verification (AINP-02)**
    1. Plug in your MIDI keyboard (if available)
    2. Click the "Start" button
    3. Play several notes across different octaves (low, middle, high)
    4. Verify: corresponding keys on the keyboard strip light up in gold when pressed and return to normal when released
    5. Verify: debug panel shows NoteOn events when keys are pressed, NoteOff events when released
    6. Verify: events show source "midi", correct note names (e.g., "C4" for middle C), timestamps
    7. Verify: no stuck keys — all notes clear when you release them

    **Step 3: Microphone verification (AINP-01)**
    1. Click Start (if not already clicked) — browser should request microphone permission
    2. Grant microphone access
    3. Play single notes on your acoustic piano, holding each for 1-2 seconds
    4. Verify: the correct key lights up on the keyboard strip for each note you play
    5. Verify: debug panel shows NoteOn events with source "mic", frequency in Hz, clarity percentage
    6. Verify: NoteOff fires after you stop playing (may take ~50ms after note decays)
    7. Test across the range: play low notes (A0-C3), middle notes (C3-C5), high notes (C5-C8)
    8. Note any octave errors (e.g., keyboard shows C5 when you played C4) — especially in the bass range

    **Step 4: Unified event verification (AINP-03)**
    1. If you have both MIDI and acoustic piano available: play the same note on both
    2. Verify: debug panel shows events from both sources with the same note name
    3. Verify: keyboard strip responds identically to both input sources
