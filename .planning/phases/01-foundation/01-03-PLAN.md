    This is a human verification checkpoint. No code changes are made.

    What was built: Complete Phase 1 foundation — React + Vite project with game canvas (L-shape path, placeholder enemies), 88-key keyboard strip, debug panel, microphone pitch detection via pitchy, and MIDI keyboard input via Web MIDI API. Both sources dispatch unified NoteOn/NoteOff events through a Zustand store.

    Present the verification steps below to the user and wait for their response.

  <verify>
    Open http://localhost:5173 in Chrome.

    **Step 1: Visual layout check**
    1. Confirm the layout fills the browser window: game canvas on the left/center, debug panel on the right, keyboard strip at the bottom
    2. Confirm the game canvas shows an L-shape path with colored circles containing note names (e.g., "C4", "E4")
    3. Confirm the keyboard strip at the bottom shows 88 keys with correct black/white key pattern
    4. Resize the browser window — confirm the canvas resizes smoothly without blur or scrollbars

    **Step 2: MIDI keyboard test** (skip if no MIDI keyboard available)
    1. Connect your MIDI keyboard via USB
    2. Press a key on the MIDI keyboard
    3. Confirm the corresponding key on the keyboard strip lights up in cyan
    4. Release the key — confirm the highlight disappears immediately
    5. Check the debug panel — confirm it shows: "midi NoteOn [noteName]" and "midi NoteOff [noteName]"
    6. Press multiple keys — confirm all active keys highlight simultaneously

    **Step 3: Microphone test**
    1. Click the "Enable Microphone" button (or similar activation prompt)
    2. Grant microphone permission when the browser prompts
    3. Play a single note on your acoustic piano (try middle C / C4 first)
    4. Confirm the corresponding key on the keyboard strip lights up
    5. Confirm the debug panel shows: "mic NoteOn C4" with frequency (~261.6 Hz) and confidence (percentage)
    6. Stop playing — confirm the highlight disappears (NoteOff event appears in debug panel)
    7. Test across the piano range: play a low note (A1 or A2), a middle note (C4), and a high note (C6 or C7)
    8. Check response speed — the highlight should appear within a fraction of a second of striking the key (<100ms goal)

    **Step 4: Unified event verification**
    1. Play a note via MIDI, then the same note via acoustic piano
    2. Check the debug panel — both should show the same note name, but with different sources (midi vs mic)
    3. Confirm both event types have: type (NoteOn/NoteOff), note name, source, timestamp

    **Report any issues:**
    - Low notes not detected (which notes?)
    - Excessive latency (estimated delay?)
    - Stuck notes (highlight stays on after release?)
    - Wrong note detected (what was played vs what was shown?)
    - Layout issues (what doesn't look right?)
  </verify>
