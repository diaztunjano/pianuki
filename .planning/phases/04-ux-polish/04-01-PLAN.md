**useAudioInput.ts — Apply latencyOffsetMs to NoteOn timestamps:**

In the `poll()` function, read the offset from the store at the top of each poll iteration:

```typescript
const latencyOffsetMs = useBoundStore.getState().settings.latencyOffsetMs
```

Then, for EVERY `performance.now()` call that stamps an event timestamp (both NoteOn and NoteOff), subtract the offset:

```typescript
timestamp: performance.now() - latencyOffsetMs,
```

There are 4 places in the poll function that call `performance.now()` for timestamps:
1. Silence-triggered NoteOff (line ~89)
2. Note transition NoteOff (line ~111)
3. New note NoteOn (line ~123)
4. Clarity-below-threshold NoteOff (line ~135)

Apply the offset in ALL 4 places. Read `latencyOffsetMs` once at the top of `poll()` and reuse the value for consistency within a single frame.

IMPORTANT: Apply offset to ALL event timestamps (NoteOn AND NoteOff), not just NoteOn. This keeps the entire event timeline consistently shifted. A positive latencyOffsetMs means "my audio system adds delay, so shift timestamps back in time to compensate."
