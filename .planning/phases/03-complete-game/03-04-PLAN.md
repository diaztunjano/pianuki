**SettingsPanel.tsx:**

A modal overlay component that receives an `onClose` callback prop.

Props: `{ onClose: () => void }`

Layout:
- Fixed overlay: `absolute inset-0 flex items-center justify-center bg-black/60 z-25` (above GameOverlay z-20, below mic overlay z-30)
- Card: same styling as GameOverlay cards (rounded-2xl bg-gray-900/90 border border-white/10 px-12 py-10 text-white shadow-2xl backdrop-blur-sm)
- Title: "SETTINGS" in bold tracking-widest

Sections:

1. **Difficulty / Penalty Mode**:
   - Label: "Difficulty"
   - Three buttons in a row, styled as a segmented control:
     - "Easy" (no penalty) — description below: "No penalty for missed enemies"
     - "Normal" (HP damage) — description: "Enemies deal damage"
     - "Hard" (enemy advances) — description: "Enemies loop back"
   - Active button highlighted (bg-white/20 border-white/40), inactive (bg-white/5 border-white/10)
   - On click: `setPenaltyMode('easy' | 'normal' | 'hard')`

2. **Input Source**:
   - Label: "Input Source"
   - Two buttons: "Microphone" / "MIDI"
   - Same segmented control style
   - On click: `setInputSource('mic' | 'midi')`

3. **Close button**: "Done" primary button at bottom → calls `onClose()`


**StatsScreen.tsx:**

Replaces the stub in AppShell.tsx. Full stats page accessible from level select.

Layout:
- Full height flex-1 scrollable container with padding
- Header: "STATS" title + "Back" button → `navigateTo('levelSelect')`

Sections:

1. **Summary row** (top):
   - Total levels completed: count of `progress.levels` entries where `completed === true`
   - Total play time: format `progress.totalPlayTimeMs` as hours/minutes (e.g. "12m" or "1h 23m")
   - Overall accuracy: average of all completed levels' bestAccuracy (or "—" if none)

2. **Per-level accuracy** (bar chart using CSS divs):
   - For each of the 5 levels, show a horizontal bar:
     - Label: level name from LEVEL_CONFIGS
     - Bar width: `bestAccuracy%` of container width (0-100%)
     - Color: green for >= 75%, yellow for >= 50%, red for < 50%, gray if not played
     - Text inside bar: `{bestAccuracy}%`
     - Stars shown next to bar: ★★★ with earned ones filled
   - Unplayed levels show empty bar with "Not played" text

3. **Most missed notes** (top 5):
   - Read `progress.noteMissCounts`, sort descending by count
   - Show top 5 as a list: "C#4 — 12 misses", "F4 — 8 misses", etc.
   - If no misses recorded, show "No data yet"

4. **Reset progress** button at bottom:
   - Secondary/danger style (red-ish)
   - On click: confirm dialog (window.confirm), then call `resetProgress()`

Selectors:
```tsx
const progress = useBoundStore((s) => s.progress)
```
